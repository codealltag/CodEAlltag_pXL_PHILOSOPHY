Dieses Mass bezieht sich auf die Wahrscheinlichkeit des Vorkommens von
Zeichen. Und somit misst man mit diesem Mass etwas völlig anderes, wenn
man vor einer Dekodierung misst als hinterher, schliesslich handelt es

eine zweite. Die zweite hat dann eben ggf. einen anderen Zeichenvorrat
und darin auch andere Verteilung.

Dieses Mass, was man besser "Informationsmenge" nennt, besagt rein gar
nichts über den Inhalt einer Nachricht.


Prinzipiell ist ein Code eine Vereinbarung für eine formale Sprache. In
einem erweiterten Code-Begriff ist er aber auch einfach eine formale
Sprache, die mit ihren Worten jeweils feste Information expliziert.
Bitte beachte dabei, dass jetzt ein informations-Begriff zum Tragen
kommt, der bedeutungsfrei gemeint ist, ohne Subjektbezug.

Ist ein solcher Code kreiert, so ist der erweiterte Code-Begriff mit dem
vorherigen der Vereinbarung kompatibel. Ist er nicht kreiert, sondern
kommt natürlich vor, so ist er trotzdem wohldefiniert.

So ist zu verstehen, wenn man von einem Code in der Genetik spricht.

Mit einer Kodierung kann man noch etwas anderes meinen: so wird auch das
Darstellen einer formalen Sprache mittels physikalischer Eigenschaften
benannt, also z.B. elektrische Signale oder aber auch Elementarmagnete,
mithin die Verstofflichung von formaler Sprache.

Um das ganz vollständig zu bekommen: natürliche Sprache lässt sich in
elektronischem Text fassen, indem man über eine Zeichenkodierung Text in
eine formale Sprache abbildet. In dieser sind deren Worte gerade die
einzelnen Zeichen, z.B. die Buchstaben, und (fast) jede Kombination
erlaubt. z.B. schreibe ich diesen Text, indem ich ihn Unicode-kodiere.

Diese Unicode-Kodierung, in der die Worte übrigens "Code Points"
heissen, und nicht genau jeweils ein Zeichen abbilden (sondern nur
meistens), kodiere ich dann weiter in eine zweite Kodierung. Diese nennt
sich UTF-8. Es ist eine Kodierung, die Unicode in 8bit-Bytes abbildet,
das ist die übliche Grösse für Bytes. Genau genommen sind 8bit-Bytes
selber eine Kodierung, wie man Zahlen von 0 bis 255 jeweils in 8 bit
kodiert, und die Kodierung danach ist physischer Natur. Sie ist
keinesfalls so einfach, dass man dann eben jeweils ein Bit nehmen
könnte, und da ein Flip-Flop drauf abbilden (so heissen die Schaltungen,
die ich hier benutze), sondern das ist komplizierter, weil ich DRAM
habe. Genau genommen ist das eine weitere Kodierung, bis wir endlich bei
den Signalen landen, die in dem Rechner, den ich benutze, durch
elektromagnetische Wellen repräsentiert werden.

Das ist das, was Kodierung ausmacht: es handelt sich um einen Code in
einem Code in einem Code in einem Code. Ein wenig wie die Matroschka.
