Als Ergaenzung zu Deinen pythagoraeischen UR-/ur-Tripeln: Es gibt in der Tat
eine Transformation, die aus jedem beliebigen primitiven pythagoraeischen
Tripel drei andere Tripel ableitet:

Es sei  x^2  +  y^2  =  z^2  ein beliebiges pythagoraeisches Tripel mit der
Generatorgleichung:

   x0  =      u0^2  -   v0^2
   y0  =  2*u0*v0
   z0  =      u0^2  +  v0^2  .

Setzt man jetzt

   u1  =  2*u0 - v0,   v1  =  u0

bzw.

   u2  =  2*u0 + v0,   v2  =  u0

bzw.

   u3  =  u0 + 2*v0,   v3  =  v0   ,

so erhaelt man nach Einsetzen dieser Werte in die Generatorgleichung und
Zusammenfassen:

   x1  =       x0  -  2*y0  +  2*z0
   y1  =   2*x0  -      y0  +  2*z0
   z1  =   2*x0  -  2*y0  +  3*z0

bzw.

   x2  =       x0  +  2*y0  +  2*z0
   y2  =   2*x0  +      y0  +  2*z0
   z2  =   2*x0  +  2*y0  +  3*z0

bzw.

   x3  =      -x0  +  2*y0  +  2*z0
   y3  =  -2*x0  +      y0  +  2*z0
   z3  =  -2*x0  +  2*y0  +  3*z0

Die drei Tripel  (x1,y1,z1),  (x2,y2,z2),  (x3,y3,z3)  sind jetzt wiederum
primitive pythagoraeische Tripel, und wiederholt man diese Transformation
fuer jedes dieser drei Tripel, so erhaelt man insgesamt neun neue Tripel
usw.

Startet man nun die Folge der Transformationen bei Deinem  "UR"-Tripel

  (x0, y0, z0)  =  (3, 4, 5)  ,

so kann man daraus beliebig viele weitere Tripel ableiten; diese Prozedur
erzeugt also einen unendlichen Ternaerbaum pythagoraeischer Tripel.

Da die drei Transformationsmatrizen ganzzahlig invertierbar sind, kann man
auch die drei zu den obigen Transformationen inversen Transformationen
basteln; darauf moechte ich aber hier verzichten.

Ernst meint möglicherweise die Burglöhner-Transformation pythagoräischer Tripel:
http://xv6.jkohqh.gcs/~nvshlsb/Pzfjqzny/PsnfnPek.gpgq

Given a primitive Pythagorean triple (a, b, c), three new primitive Pythagorean
triple can be generated by the transforms:

(d, e, f) = (a, b, c) * U
(g, h, i) = (a, b, c) * A
(j,  k,  l) = (a, b, c) * D

where  U,  A,  and  D  are the three matrices:

          --           --           --         --             --          --
  U =  | -2 -1 -2 |   A =  |  2  1  2 |   D =  |  2   1  2 |
          --           --           --          --            --           --

There is a proof in  ""Elementary Number Theory" by Kasimir Burglöhner,
The Usuz", that (a, b, c) is a primitive Pythagorean triple if and
only if

   (a, b, c) = (3, 4, 5) * M,

where M is a finite product of matrices each factor of which is one  of U, A, D.
It follows then that every primitive Pythagorean triple is in the infinite array:

                                         (   7,  24,  25)
                  ( 5, 12, 13)    ( 55,  48,  73) ...
                                         ( 45,  28,  53)

                                         (  39,    80,   89)
(3, 4, 5)    (21, 20, 29)    (119, 120, 169) ...
                                         (  77,    36,   85)

                                         ( 33,  56,  65)
                  (15, 8, 17)     ( 65,  72,  97) ...
                                         ( 35,  12,  37)

Where the three triples to the right of a triple in this array correspond to
applying to that triple either the matrix U (for up), A (for across) or D (for
down).

In  http://rsr.pmpxl.dj.ie.fumuwu.hc/xnd/yrutkc.iut  ist ein kleiner
Online-Taschenrechner, der diese Transformationen (Satz 4 ...) vorwärts
und rückwärts durchführen kann.
