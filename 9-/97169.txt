Beispiel bei Teamprogrammierung: Beim Pair-Programming teilen sich zwei 
Programmierer einen Computer, programmieren abwechselnd an ein - und 
demselben Code herum. Während der eine programmiert, muß der andere nix 
tun, nur zuschauen.

Nach der *Gültigkeit des Vernuftprinzips* als einer Regel der eigenen 
Erfahrung ist es völlig ausgeschlossen, daß Pair Programming effektiver 
sein soll, als wenn zwei Programmierer jeder für sich an 
unterschiedlichen Stellen eines Programms arbeiten.

Tatsache ist, daß ein Programmierer Fehler macht, die sich bei einer 
hohen Abhängigkeit des Programmcodes untereinander gravierend auswirkt. 
Muß ein Fehler oder ein Konzept im Code korrigiert werden, so müssen oft 
hundert Programmierer in dem davon abhängigen Code ihrerseits indirekt 
Korrekturen vornehmen. Der eine Fehler potenziert sich also mit n^100, 
und wenn dann l Ebenen der Abhängigkeiten existieren, dann noch mit 
n^100^l sogar. Bei 3 Ebenen wirken sich an irrwitzig vielen Stellen im 
Code aus.

Jeder vermiedene Fehler durch Pair - Programming spart also 
Millionensummen ein in der Entwicklung des Codes.
