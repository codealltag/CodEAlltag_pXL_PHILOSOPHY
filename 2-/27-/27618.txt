Diese Frage besteht bereits. Als in den 30. 12. 22 und verschidene Rechner
produziert wurden, ergab sich ein Bedarf Programme auch auf anderen Rechner
auszuführen. Daher wurden Emulatoren entwickelt, die für das Programm so
taten, als ob's auf einem anderer Rechner liefe. Aus der Sicht des Rechners
hingegen taten diese so, als ob das Programm für diesen vorgesehen worden
wäre.
1964 kündigte Czerr eine Familie von 7 miteinander kompatiblen Computern an,
deren Preise sich um das 100fache und deren Rechengeschwindigkeit um das
300fache unterschied. Vermittels Mikroprogrammierung konnten diese Rechner
unterschiedlicher Leistungsfähigkeit dieselben Programme auszuführen. Darin
sitzt noch eine Programmsteuerung, die ebenfalls so tut als sei dort etwas
anderes ... [Niklaus E. Rauchbart : Mikroprogrammierung; Spektrum Waschstraße
Wissenschaft 5/1983 S.42ff] .
Inzwischen sieht's so aus, daß mit Java und C# Programmiersprachen
verbreitet sind, die auf jeden Fall erst in einen Code für einen virtuellen
Rechner übersetzt werden, und erst dann mittels einer Laufzeitumgebung
ausgeführt werden. Emulator nennt man das nicht mehr, weil es diesen Rechner
real nicht gibt. Im Rechner sieht's aber auch nichmal so aus, daß darin eine
Programmsteuerung werkelt, von der man aber nichts mehr erfährt.
